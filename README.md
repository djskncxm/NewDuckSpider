# DuckSpider

一个面向 Go 语言的轻量级爬虫框架，**在用户侧刻意贴近 Scrapy 的编程模型**，用于降低 Python 爬虫用户迁移到 Go 的成本。

DuckSpider 并不追求功能完备，而是希望在保证可用性的前提下，让代码尽量少、结构尽量清晰，适合作为学习和二次开发的基础。

---

## Why

目前 Go 的爬虫生态在「工程化」和「学习友好度」之间往往需要取舍：

- 工程化框架通常结构复杂，不适合初学者阅读
- 简单示例又很难支撑真实项目

DuckSpider 的初衷是：

- 为 Go 提供一种 **Scrapy 风格的爬虫开发体验**
- 降低从 Python 爬虫迁移到 Go 的心智负担
- 用尽量少的代码实现一个**完整可跑的爬虫框架**

这个项目最初服务于作者自己的爬虫项目，同时也作为对爬虫框架架构的学习与实践。

---

## Design Philosophy

- **Scrapy-like API**  
  用户侧编程模型参考 Scrapy，但不照搬其实现细节

- **Small & Readable**  
  代码量刻意控制，推荐直接阅读源码理解整体架构

- **Explicit over Magic**  
  尽量避免隐式行为，流程清晰可追踪

- **Learning-oriented**  
  既能用，也适合拿来拆解、修改、重构

---

## Non-Goals

DuckSpider **不打算**：

- 完整复刻 Scrapy 的功能体系
- 提供复杂的插件系统或中间件机制
- 覆盖所有反爬与分布式场景
